01 Security

Mutual TLS (mTLS)

Automatic encryption for all service-to-service communication. Zero-trust security without changing a single line of application code.

  • STRICT vs PERMISSIVE mTLS modes
  • Certificate lifecycle managed by istiod
  • Block non-mesh traffic automatically
  • Inspect TLS certificates in real-time
▶ chapter-01-mtls\startup.bat
02 Traffic

Canary Deployments

Gradually shift traffic from v1 to v2 using weighted routing. Roll out new versions safely with fine-grained percentage control.

  • 90/10 → 50/50 → 100% traffic shifting
  • Weighted routing via VirtualService
  • DestinationRule subsets for version targeting
  • Safer than Kubernetes rolling updates
▶ chapter-02-canary\startup.bat
03 Traffic

Intelligent Traffic Routing

Route traffic based on HTTP headers, user identity, and custom conditions. Enable A/B testing and dark launches without code changes.

  • User-identity routing (login as 'jason')
  • Custom header-based routing (x-canary)
  • Match rule priority and ordering
  • Feature flags via infrastructure
▶ chapter-03-traffic-routing\startup.bat
04 Resilience

Fault Injection

Inject delays and HTTP errors to test application resilience. Discover hidden bugs like timeout misconfigurations through chaos engineering.

  • 7-second delay injection on ratings service
  • HTTP 500 abort injection
  • Discover timeout bugs between services
  • No application code changes needed
▶ chapter-04-fault-injection\startup.bat
05 Resilience

Circuit Breaking

Prevent cascade failures by limiting connections and ejecting unhealthy endpoints. The circuit breaker pattern protects your entire mesh.

  • Connection pool limits (maxConnections)
  • Outlier detection and host ejection
  • Load testing with Fortio
  • Observe 503 overflow responses
▶ chapter-05-circuit-breaking\startup.bat
06 Resilience

Rate Limiting

Control traffic rates with local rate limiting using EnvoyFilter. Protect your services from overload with the token bucket algorithm.

  • 10 requests/minute limit on productpage
  • Token bucket algorithm in action
  • 429 Too Many Requests responses
  • Custom response headers for debugging
▶ chapter-06-rate-limiting\startup.bat
07 Observability

Kiali Dashboard

Visualize your entire service mesh topology in real-time. See traffic flows, health status, and Istio configuration validation.

  • Real-time service mesh graph
  • Traffic animation and flow visualization
  • Health indicators and error detection
  • Istio configuration validation
▶ chapter-07-kiali-dashboard\startup.bat
08 Observability

Grafana & Prometheus

Monitor your mesh with automatic metrics collection. Explore pre-built Istio dashboards and learn PromQL queries for deep insights.

  • Automatic Envoy metrics collection
  • Pre-built Istio Grafana dashboards
  • PromQL queries for Istio metrics
  • The 4 Golden Signals monitoring
▶ chapter-08-grafana-prometheus\startup.bat
09 Observability

Distributed Tracing (Jaeger)

Trace requests across microservices with Jaeger. Understand call chains, latency breakdowns, and debug performance issues.

  • Automatic trace header injection
  • Jaeger UI for trace visualization
  • Span timing and parent-child hierarchy
  • Cross-service latency analysis
▶ chapter-09-jaeger-tracing\startup.bat
10 Traffic

Traffic Mirroring

Send shadow copies of live traffic to a test service. Validate new versions with real production traffic — zero risk to users.

  • Mirror 100% of traffic to reviews-v3
  • Fire-and-forget (responses discarded)
  • Verify via pod logs comparison
  • Dark launching for safe testing
▶ chapter-10-traffic-mirroring\startup.bat

🔭 Bookinfo Application Architecture

Browser / User | v [Istio Ingress Gateway] ---- Handles external traffic | v [productpage] (Python, v1) ---- Frontend service / \ / \ v v [details] [reviews] ---- Backend services (Ruby) (Java) | | | v1 v2 v3 | | v v [ratings] ---- Rating data (Node.js) Each service has an Envoy sidecar proxy injected by Istio All traffic flows through these sidecars = full control
